# Blog Rewrite Docs

这组文档从旧的 `checkin-app-frontend` 评估中整理而来，当前作为 `blog` 新项目的重构参考基线。

建议阅读顺序：

1. [01-current-assessment.md](./01-current-assessment.md)
2. [02-rebuild-architecture.md](./02-rebuild-architecture.md)
3. [03-migration-roadmap.md](./03-migration-roadmap.md)

如果你只看一份，先看 `02-rebuild-architecture.md`。

当前文档重点解决这些问题：

- 旧项目是否值得继续维护
- 新项目前后端该怎么拆
- 数据库怎么引入
- 登录方式先做哪些最合适
- 真正的 Markdown 应该怎么保存和渲染
- 重做时按什么顺序最稳
